## 二、API 端点清单 (28个)

### 视频解析与下载
```
POST /api/info              → 解析视频信息
POST /api/download          → 创建下载任务（需登录+权限检查）
GET  /api/downloads         → 获取下载历史（按用户过滤）
GET  /api/downloads/{id}    → 下载文件
GET  /api/task/{task_id}    → 查询任务状态
WS   /api/ws/progress/{id}  → 实时进度推送
```

### 批量操作
```
POST /api/batch-info        → 批量解析（最多10个）
POST /api/batch-download    → 批量下载
```

### 认证系统
```
POST /api/auth/send-code    → 发送验证码
POST /api/auth/login        → 登录获取Token
GET  /api/auth/me           → 获取用户信息
GET  /api/auth/check-permission → 检查下载权限
```

### Cookie 管理
```
POST   /api/cookies         → 上传Cookie
GET    /api/cookies         → 查询Cookie状态
DELETE /api/cookies/{platform} → 删除Cookie
```

### 云盘同步
```
GET  /api/cloud/status      → 云盘连接状态
GET  /api/cloud/auth-url    → 获取授权URL
POST /api/cloud/connect     → 交换授权码
POST /api/cloud/upload      → 上传文件到网盘
```

### AI 工具
```
GET  /api/ai/capabilities   → AI能力列表
POST /api/ai/subtitle       → 生成字幕
POST /api/ai/super-resolution → 超分增强
POST /api/ai/watermark-removal → 去水印
```

## 三、发现的问题

### 🔴 高优先级

| # | 问题 | 位置 | 影响 |
|---|------|------|------|
| 1 | JWT Secret 硬编码 `snapvid_secret_key_2026` | auth.py | 安全风险 |
| 2 | Token 通过 query string 传递 | routes.py | 安全隐患(URL日志泄露) |
| 3 | 无 API 速率限制 | routes.py | 可被滥用 |
| 4 | Admin 固定验证码 000000 | auth.py | 生产环境必须改 |

### 🟡 中优先级

| # | 问题 | 位置 | 影响 |
|---|------|------|------|
| 5 | 下载历史存内存，重启丢失 | downloader.py | 用户体验 |
| 6 | 无文件 TTL 自动清理 | 下载目录 | 磁盘空间 |
| 7 | AI/云盘/SMS 为演示模式 | services/ | 功能不完整 |
| 8 | 无分页查询 | routes.py | 大量数据时性能 |

### 🟢 低优先级

| # | 问题 | 位置 | 影响 |
|---|------|------|------|
| 9 | 无国际化(i18n) | 前端 | 海外市场 |
| 10 | 无数据统计面板 | 缺失 | 运营分析 |
| 11 | 定价按钮全禁用 | PricingPanel | 变现路径 |
| 12 | 抖音需 Cookies 才能解析 | yt-dlp限制 | 平台兼容性 |

## 四、架构评价

### 优点
- ✅ 全栈一体化，Docker 一键部署
- ✅ 合规意识强，版权过滤机制到位
- ✅ 用户体系完整（认证+授权+配额）
- ✅ UI 设计精致（深色极简+动画）
- ✅ 代码结构清晰（services 分层）
- ✅ 功能覆盖全面（30+ 下载参数）

### 改进方向
- ❌ 数据持久化不足（需 Redis/PostgreSQL）
- ❌ 安全审计未完成（密钥管理）
- ❌ 附加功能为演示态（AI/云盘/支付）
- ❌ 缺少监控和日志系统
- ❌ 缺少自动化测试
